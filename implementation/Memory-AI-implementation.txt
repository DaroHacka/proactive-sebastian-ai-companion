MEMORY-AI-IMPLEMENTATION
========================

Date: 2026-04-23

## Vision

Instead of saving full conversation transcripts, prompt the AI after conversation ends 
(or after 30-40 min inactivity) to generate a summary of the conversation and save 
that as memory.

## Current Implementation (NOT Implemented)

Currently, memories are saved as full conversation transcripts:
- User message + AI response saved to memory/fresh.json
- Entries move from fresh → medium → longterm as counts grow
- No AI summarization happening

## Proposed Implementation

### Flow

```
Conversation ends (or 30 min inactivity)
        ↓
Script prompts AI: "Summarize our conversation briefly"
        ↓
AI outputs summary: "Elias is working on an AI companion project..."
        ↓
Save SUMMARY as memory (not full transcript)
```

### When to Trigger

| Trigger | Timing |
|---------|--------|
| **Inactivity** | After 30-40 min since last message |
| **Session End** | When conversation naturally ends |

### Storage Format

Replace current structure:
```json
{"user_message": "...", "ai_message": "..."}
```

With:
```json
{
  "user_message": "...",
  "ai_message": "...", 
  "summary": "Elias is building an AI companion...",
  "key_facts": ["Elias: working on Sebastian project", "Mood: enthusiastic", ...]
}
```

### Prompt Example

```
Summarize our recent conversation in 2-3 sentences. Focus on:
- Main topics discussed
- Any plans or commitments made
- Elias's mood or concerns
- Important things to remember about Elias

Format your response as a brief paragraph.
```

### Memory Tiers

| Tier | Content | Count |
|------|---------|-------|
| Fresh | Last 10 summaries | Recent context |
| Medium | 11-50 summaries | Medium-term |
| Long-term | 51+ summaries | Persistent facts |
